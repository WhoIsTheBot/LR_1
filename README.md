1. Що таке задача?У контексті .NET задача — це одиниця роботи, яка представлена класом Task і виконується асинхронно, зазвичай у фоновому потоці. Вона дозволяє програмі виконувати кілька операцій одночасно, не блокуючи основний потік виконання.  

2. Чим відрізняється виконання багатопоточної програми від однопоточної?В однопоточній програмі інструкції виконуються послідовно одна за одною, що може призвести до очікування завершення тривалих операцій. Багатопоточна програма дозволяє виконувати кілька завдань паралельно, що підвищує ефективність, проте вимагає використання об'єктів синхронізації для уникнення хаотичного виводу даних або помилок.  

3. Як створити і запустити задачу?Задачу можна створити та запустити за допомогою статичного методу Task.Run(), передавши в нього дію (наприклад, метод Update), яку потрібно виконати. Також можна ініціалізувати об'єкт Task і викликати метод Start(), щоб поставити задачу в чергу на виконання.  

4. Як завершити потік?Потік автоматично завершується після виконання коду в методі, який йому було передано (наприклад, методу Run). Основний потік може очікувати завершення фонового потоку за допомогою методу Join(), який блокує виконання до моменту повної зупинки дочірнього процесу.  

5. Як передати аргумент потоку?Аргументи можна передати при створенні об'єкта потоку (наприклад, через параметризований конструктор або лямбда-вираз) або при запуску процесу через властивість Arguments об'єкта StartInfo. У наведених прикладах аргументи часто передаються у вигляді імені об'єкта синхронізації (м'ютекса) для координації дій.  

6. Пріоритети потоків. Як поміняти пріоритет потоку?Хоча у наданому тексті детально не описано механізм зміни пріоритетів, у системі .NET це робиться через властивість Priority об'єкта класу Thread. Вона дозволяє вказати рівень доступу до процесорного часу (наприклад, Highest або Lowest), проте за замовчуванням усі потоки мають рівень Normal.  

7. Який метод в новому потоці запускає метод Start() класу Task?Метод Start() ініціює виконання делегата (методу), який був переданий задачі при її створенні. Після цього планувальник завдань викликає відповідний користувацький метод (наприклад, Update або Run) у вільному фоновому потоці.  

8. Що робить метод Wait() класу Task?Метод Wait() зупиняє виконання поточного потоку до тих пір, поки задача (об'єкт Task), для якої його викликано, не завершить свою роботу. Це гарантує, що наступні операції будуть виконані лише після отримання результату асинхронної дії.
